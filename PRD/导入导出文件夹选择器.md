# PRD：导入/导出功能改用原生文件夹选择对话框

## 1. 背景

RoxyBrowser Manager 的"导出配置"和"导入配置"功能目前采用**文本输入框**让用户手动输入文件夹路径。该方式在 Windows 平台上体验较差：

- 用户需要手动拼写或复制粘贴完整路径
- `~/Desktop/...` 格式在 Windows 上无法被正确识别
- 容易因输入错误导致操作失败

## 2. 目标

将导入和导出的路径选择方式，从**文本输入模态框**改为**操作系统原生文件夹选择对话框**，提升用户体验。优先适配 Windows 平台。

## 3. 功能需求

### 3.1 导出配置

| 项目 | 描述 |
|------|------|
| 触发方式 | 点击设置菜单中的"导出配置" |
| 交互流程 | 直接弹出 Windows 原生"选择文件夹"对话框 |
| 成功行为 | 用户选择文件夹后立即执行导出，完成后弹出成功提示 |
| 取消行为 | 用户关闭对话框则不做任何操作 |
| 错误处理 | 如导出失败，弹出包含错误信息的提示 |

### 3.2 导入配置

| 项目 | 描述 |
|------|------|
| 触发方式 | 点击设置菜单中的"导入配置" |
| 交互流程 | 直接弹出 Windows 原生"选择文件夹"对话框 |
| 成功行为 | 用户选择包含 `config.json` 与 `profiles` 目录的文件夹后执行导入 |
| 取消行为 | 用户关闭对话框则不做任何操作 |
| 错误处理 | 如目录中缺少 `config.json`，提示"导入目录中未找到 config.json" |

### 3.3 对话框标题

- 导出时：`选择导出目录`
- 导入时：`选择导入配置目录`

## 4. 非功能需求

- 项目已集成 `tauri-plugin-dialog`，本次需求复用现有插件能力
- 后端导入/导出逻辑（`export_profiles` / `import_profiles`）无需改动
- 前端删除不再使用的模态框代码，减少冗余

## 5. 影响范围

| 文件 | 变更类型 |
|------|----------|
| `src-tauri/src/commands/settings.rs` | 新增 `browse_for_folder` 命令 |
| `src-tauri/src/lib.rs` | 注册新命令 |
| `src/components/Dashboard.tsx` | 删除模态框，改用对话框调用 |

## 6. 验收标准

- [ ] 点击导出配置后弹出 Windows 原生文件夹选择器（非文本输入框）
- [ ] 选择文件夹后导出成功，取消则无操作
- [ ] 点击导入配置后弹出 Windows 原生文件夹选择器
- [ ] 选择有效导出目录后导入成功
- [ ] 选择无效目录后提示明确错误信息
- [ ] 无残留的模态框 UI 代码
